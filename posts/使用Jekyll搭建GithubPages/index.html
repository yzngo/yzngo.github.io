使用Jekyll搭建GithubPages

By Yzngo
Posted 2019-04-16 Updated 2022-07-09 3 min read

记录Github Pages搭建流程，以备以后查阅。

资源网站

  1. Jekyll官方主题站

安装Jekyll

跟随Jekyll Docs安装 Ruby, RubyGems, Jekyll, Bundler

  • 具体步骤简单记录如下

    1. 安装 RubyInstaller -> Ruby+Devkit 3.1.1-1 (x64)
    2. 在安装向导的最后一步， 执行 ridk install
    3. 在自动打开的CMD窗口中，RubyInstaller2 安装选项选择 3
    4. 安装完成后打开一个新的终端，运行 gem install jekyll bundler
    5. 检查 Jekyll 是否安装成功 jekyll -v

创建/更新Github Pages

下面的步骤基于主题 Chirpy 。

  1. 登录Github，通过 Chirpy Starter 创建新的repo, 命名为[username].github.io

  2. 把此repo Push到本地。

预览

  • 在仓库根目录运行
    • Bundle
    • bundle exec jekyll s
  • 浏览器打开 http://127.0.0.1:4000

部署（仅第一次）

  • 检查 _config.yml中的 url 是否配置成 [username].github.io

  • 确保 .github/workflows/pages-deploy.yml 文件存在

    • 其中 on.push.branches 应该和默认分支相同

  • 确保 tools/deploy.sh 存在

  • 更新lock file

    1

      bundle lock --add-platform x86_64-linux

  • Push and commit

    • 等待 GitHub Actions 运行完
    • 会增加一个新的分支 gh-pages

发布

  • GitHub仓库设置页面： repo -> Settings -> Pages

    • 修改Source为：gh-pages /(root)
    • 保存

配置博客

  1. 大多数配置都在 ./config.yml 根据注释配置即可
    • 重点需要关注：url，avatar，timezone，lang
  2. 配置favicon，参考： 2019-08-11-customize-the-favicon.md
  3. 启用GooglePV，参考：2021-01-03-enable-google-pv.md

文章格式

  1. 所有文件放到 _posts 文件夹下，子文件夹可以随意建立，对blog透明
  2. 文件名约定格式 YYYY-MM-DD-TITLE.md
  3. 网站上的文件夹， Tags 会根据文章中的元数据划分
  4. Jekyll-Compose - 一个文件生成工具，方便填写元数据

### 可选
pin: true								# 是否置顶，默认false
toc: false								# 是否显示目录，默认true
math: true								# 是否启用数学公式，默认false
mermaid: true							# 是否启用Mermaid图表，默认false
comments: false							# 是否开启评论 
image:									# 页首y图片
	path: 								# 图床路径，不能空
	w: 1000								# in pixels，不能空
	h: 400								# in pixels，不能空
	alt: This is an image.	  			# 替代文本，可空
author: <author_id>						# 如果在 _data/authors.yml中设置了作者信息
---

作者信息

  1. 作者信息默认会从 _config.yml 中提取。

    1
2

     social.name
 social.links

  2. 也可以自定义，新建文件 _data/authors.yml, 填入以下内容

    1
2
3
4

     <author_id>:
   name: <full name>
   twitter: <twitter_of_author>				# good for SEO
   url: <homepage_of_author>

Upgrading

  1. 修改 Gemfile 更新主题版本

    1
2

     - gem "jekyll-theme-chirpy", "~> 3.2", ">= 3.2.1"
 + gem "jekyll-theme-chirpy", "~> 3.3", ">= 3.3.0"

  2. 执行以下命令

    1

     $ bundle update jekyll-theme-chirpy

  3. 更新内容查看
