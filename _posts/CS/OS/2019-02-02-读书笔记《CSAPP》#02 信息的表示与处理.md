# 二 信息的表示与处理

1. 计算机中所有的信息最终都表示为bit的组合.
2. 把bit组合起来(8bit, 64bit), 并赋予*不同的可能位模式*以含意, 我们就能够表示任何有限集合的元素
        1) 二进制数字系统: 整数, 浮点数...
        2) 字符码: 英文, 汉字, 控制字符...

## 2.1 信息存储
1. C中的指针都是某个存储块的第一个字节的虚拟地址.
2. C编译器将*指针*和*类型*联系起来, 就可以根据指针值的类型, 生成不同的机器级代码访问存储在指针所指向位置处的值

### 2.1.1 十六进制表示
1. 记住 A-10 [C-12] F-15
2. 快速计算2的n次幂的十六进制表示   p26
   - 把幂次n写成这种形式 n=i+4*j  i后面加j个零  
   - n的开头=1(i=0) 2(i=1) 4(i=2) 8(i=3)
   - 例 [2048=2^11] => [n=11=3+4^2] => [0x800]                   短除,按幂乘
                                                        [十进制]   <->    [十六进制] <-> [二进制]
### 2.1.2 字数据大小
1. 内存和虚拟内存都是按照字节(8bit=0xFF)编址的
2. 而地址(索引)的长度是按字编码的,即地址的长度和字长相同, 如64位机器可寻址的空间为 0 ~ 2^64-1 

### 2.1.3 字节顺序
1. 大端法(big endian): 先存高字节, 符合阅读顺序
2. 小端法(little endian): 先存低字节, 比较常见

### 2.1.4 字符串
1. UTF-8 使用1~4字节为每个字符编码, 是变长的 
    - ASCII码 1个字节
    - 中文,日文 3个字节

### 2.1.6 位级运算,逻辑运算,移位运算
1. 位级运算: [&]与 [|]或 [~]非 [^]异或(不带进位的加)
2. 位向量: 固定长度为w,由0和1组成的串       [00101010100101]
3. 位向量可以表示有限集合,能对集合编码
    位向量:                  集合:
      - a=[01101001] =>        A = {0,3,5,6} 
      - b=[01010101] =>        B = {0,2,4,6}
    - a|b=[01111101] =>   A(or)B = {0,2,3,4,5,6}          | => 并
    - a&b=[01000001] =>  A(and)B = {0,6}                  & => 交
    -  ~a=[10010110] =>   (not)A = {1,2,4,7}              ~ => 补
     可以指定一个位向量掩码,有选择的使能或者屏蔽一些信号
4. 逻辑运算: [&&]与  [||]或  [!]非
5. 移位运算
    - [x<<k]x左移k位   [x>>k]x右移k位(分逻辑右移[左端补0]和算数右移[在左端补k个最高有效位])
      - 对*有*符号数使用算数右移, 无符号数使用逻辑右移

## 2.2 整数表示

### 2.2.2 无符号(整)数的编码
1. 无符号数和位向量可以直接转换
2. [位向量<->无符号数]是一个双射

### 2.2.3 有符号(整)数的编码(补码)
1. 补码编码和无符号数的唯一区别是最高有效位的权重不同. 1->负权 0->正权
2. [位向量<->补码]也是一个双射
3. |TMin| = |TMax| + 1
    - 因为一半的位模式表示负数, 另一半的位模式表示非负数[0+正数]
4. |UMax| = 2Tmax + 1
    - 即最大的无符号数数值刚好比补码的最大值的两倍大1
5. -1和UMax位模式相同,即 111111111
6. 所有机器都使用补码表示有符号数
7. 原码和反码都*不是*双射, 分+0和-0

### 2.2.4 有符号数和无符号数之间的转换
1. 对大多数语言和实现来说, 有符号数和无符号数的强制类型转换的结果都是*保持位模式不变*, 只是改变解释这些位的方式
2. 对[0<=x<=TMax]之内的值x而言,无符号数和补码表示相同.
3. 对这个范围以为的数值, 转换需要加上或者减去 2^w

### 2.2.5 C语言中的有符号数和无符号数
1. 要创建一个无符号常量, 加后缀u或U    12345u
2. 格式化字符串 十进制有符号数[%d]   十进制无符号数[%u] 十六进制数[%x]

### 2.2.6 扩展一个数字的位表示
1. 无符号数 -> 零扩展 Zero Extension
2. 有符号数 -> 符号扩展 Sign Extension
