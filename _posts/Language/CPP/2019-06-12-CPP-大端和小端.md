## 大端和小端

1. 对于位数大于8位的处理器，例如16位或者32位的处理器，由于寄存器宽度大于一个字节，那么必然存在着一个如果将多个字节安排的问题。因此就导致了大端存储模式和小端存储模式。
2. **大端模式先存高位字节，高位字节存在低位地址上。**
3. **小端模式先存低位字节，低位字节存在低位地址上。**
4. 大端模式和字符串的存储顺序相同，符合阅读习惯。
5. x86和一般的OS（如windows，FreeBSD,Linux）使用的是小端模式。
    MacOS是大端模式。

```
int x = 0x11223344;

内存地址    大端    小端
0x0001      11      44
0x0002      22      33
0x0003      33      22
0x0004      44      11
```



## 大小端测试

```c++
short int x;
char x0, x1;
x = 0x1122;
x0 = ((char*)&x)[0]; //低地址单元
x1 = ((char*)&x)[1]; //高地址单元
```
