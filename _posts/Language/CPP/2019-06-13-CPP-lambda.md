C++11 中的 Lambda 表达式用于定义并创建匿名的函数对象，以简化编程工作。

## 基本语法结构

````c++
// ()中的为可选
[capture] (parameters) (mutable) (exception 声明) (-> return_type) {
    //statements
}
````

### 1. capture

- **值捕获**：与参数传值类似，值捕获的前提是变量可以拷贝，不同之处则在于，被捕获的变量在 Lambda 表达式被创建时拷贝，而非调用时才拷贝。

- **引用捕获**： 与引用传参类似，引用捕获保存的是引用，值会发生变化。

| 捕捉变量形式 | 捕获类型           | 说明                                                   |
| ------------ | ------------------ | ------------------------------------------------------ |
| `[]`         | -                  | 未捕获变量，试图在Lambda内使用任何外部变量都是错误的。 |
| `[x, &y]`    | 值捕获，引用捕获   | x按值捕获，y按引用捕获                                 |
| `[&]`        | **隐式按引用捕获** | 用到的任何外部变量都隐式按引用捕获                     |
| `[=]`        | **隐式按值捕获**   | 用到的任何外部变量都隐式按值捕获                       |
| `[&，x]`     |                    | x显示按值捕获，其他变量按引用捕获                      |
| `[=, &z]`    |                    | z按引用捕获，其他变量按值捕获                          |
|              |                    |                                                        |

- 只能捕获那些定义Lambda为止时Lambda所在作用范围内可见的局部变量。（包括所在类的this指针）
- 值捕获的前提是变量可以拷贝，不同之处在于，被捕获的变量在Lambda表达式被创建时拷贝，而非调用时拷贝。

#### 3. mutable/exception

- Lambda 默认是 `const` 的。
- 加上 `mutable` 修饰符后，可以修改传递进来的拷贝（注意是能修改拷贝，而不是值本身）
- exception 声明用于指定函数抛出的异常

#### 4. return_type

- 标识函数返回值的类型。
- 当返回值为 void，或函数体中只有一处 return 时，可以省略。
