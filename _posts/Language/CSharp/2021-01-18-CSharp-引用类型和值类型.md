## 引用类型和值类型

| 引用类型                                                     | 值类型                                                       |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| Pass By Reference                                            | Pass By Value                                                |
| 引用类型的实例总是在堆上创建                                 | 变量的值在它声明的位置存储(可能在堆上,可能在栈上)            |
| 引用类型必须 new 出来                                        | 值类型可以不用new，值类型的new仅仅调用构造函数初始化数据     |
|                                                              | 所有值类型都是`sealed`密封的，不能被继承，值类型只能继承接口 |
|                                                              | 值类型一般用于小数据，小于16B最好                            |
|                                                              | 一般要创建`Immutable`的值类型，一旦实例化就不要修改其值      |
| 引用类型的表达式(如一个变量)的值是一个引用                   | 值类型表达式(如一个变量)的值是实际的数据                     |
| 引用类型存取慢                                               | 值类型存取快                                                 |
| 堆内存由GC释放                                               | 栈内存自动释放                                               |
| 引用类型继承自 System.Object                                 | 值类型继承自 System.ValueType                                |
| 引用拷贝：引用类型赋值时会把**引用**拷贝一份，然后赋值给另一变量 | 值拷贝：值类型赋值时会把**真实数据**拷贝一份，然后赋值给另一个变量。 |
|                                                              |                                                              |

1. 引用类型的表达式,它的值是一个引用,而不是该引用所指代的对象.

    例如: `String.Empty`的值不是一个空字符串,而是对空字符串的引用.

    		`String.Concat`的作用是返回一个字符串(的引用), 该字符串的所有参数都连接到了一起.

2. 类型, 不管是类还是结构体, 拥有多少方法并不重要,每个实例所占用的内存不会受到影响. 虽然代码本身会消耗内存, 但只会发生一次.

3. 变量的值在它声明的位置存储.

    - 引用类型的实例总是在堆上创建.
    - 类的值类型的实例变量总是和对象中的其他数据放在一起, 也就是在堆上.
    - 局部变量和方法参数一般在栈上创建.
    - 被匿名方法捕获的变量要存在堆上.
    
4. 引用类型作为方法参数使用时,参数默认以“值类型”方式传递, 只不过值本身是一个引用.

## 装箱和拆箱

![image-202107051asdasdasd](https://cdn.jsdelivr.net/gh/yzngo/picture/img/202205162305356.png)


- Boxing: 根据值类型创建一个堆上的对象, 然后使用对这个新对象的一个引用.
- 性能问题:
    - 装箱次数太多会增大程序本身的操作开销.
    - 会创建大量的对象,加重GC的负担.
