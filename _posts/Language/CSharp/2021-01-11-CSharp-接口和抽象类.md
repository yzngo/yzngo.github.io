## 语法区别

| Interface                                        | Abstract Class             |
| ------------------------------------------------ | -------------------------- |
| 接口中不能有字段，只能有方法和属性，事件，索引器 | 抽象类无限制               |
| 接口只能包含抽象成员，不能有实现                 | 抽象类可以有               |
| 接口中所有成员不能有修饰符修饰                   | 抽象类无限制               |
| 一个类可以实现多个接口                           | 一个类只能继承一个类       |
| 实现类和接口是`can do`的关系                     | 子类和抽象类是`is a`的关系 |
|                                                  |                            |
|                                                  |                            |

1. 如果派生类没有全部override父类的所有抽象方法, 则派生类也是抽象类
2. 抽象类和普通类其实是一回事(除了不能实例化), 只是他可以添加抽象方法和抽象属性, 留给子类去实现.
3. 使用接口类型的变量只可以调用该接口定义的方法和 System.Object 的方法



## 用法区别

| Interface                                | Abstract Class                   |
| ---------------------------------------- | -------------------------------- |
| 接口定义对象的行为 `Can do`              | 抽象类抽象对象的属性 `Is a`      |
| 接口的设计目的，是**对类的行为进行约束** | 抽象类的设计目的，是**代码复用** |

1. 接口的设计目的，是对类的行为进行约束，也就是提供一种机制，可以强制要求不同的类具有相同的行为。它只约束了行为的有无，但不对如何实现行为进行限制。
2. 而抽象类的设计目的，是代码复用。当不同的类具有某些相同的行为(记为行为集合A)，且其中一部分行为的实现方式一致时（A的非真子集，记为B），可以让这些类都派生于一个抽象类。
